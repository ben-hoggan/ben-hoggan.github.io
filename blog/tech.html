What are the best practices associated with using classes vs. ids?

Introduction

Webpages are made up of "elements". An element on a webpage could be a body of text, a title, an image or anything else you might see on a website. After a web developer adds one of these elements to a page they can style it to change it's shape, location and overall look. To assign a style to an element a developer will either assign it a "Class" or an "ID".

Classes

A developer can add a class style to an element and/or multiple elements on a page. This can be especially useful when you want lots of elements to look the same. Assigning multiple elements the same class will make the visually similar.

IDs

A developer can add an ID to an element. This ID is meant to only be used on a single element on a page. This can be especially useful when you might need to single out a specific element on a page, for example to change it dynamically without changing other elements on the page.

Conclusion

Use classes to style your page. Only use an ID when you will need to interact with a single element on a page.
